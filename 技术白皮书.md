# 量化策略平台技术白皮书

> 版本：2026-06-19
> 范围：平台架构、可复用组件、策略组合与回测结果

---

## 1. 设计目标

构建一个**多股票、多 Pipeline、多策略**的量化研究与决策平台：

1. 任何预测思想都可以封装为独立的 **Model**、**Pipeline**、**Strategy**。
2. CLI 与 Web UI 共享同一套底层组件，没有重复实现。
3. 因子、目标、模型、管线、策略全部通过注册表管理，便于扩展与回测对比。
4. 通过季度分析与全区间扫描，自动为每只股票挑选当前最优的 (Pipeline, Strategy) 组合。

---

## 2. 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                     CLI / Web UI / API                       │
│   QuantCLI.bat / scripts\strategy.bat                         │
│   QuantWebUI.bat / scripts\start.bat                          │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                  Data 层（src/data/）                         │
│  load_stock(symbol) 从 data/stocks/ 加载 OHLCV CSV            │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                  Pipeline 层（src/pipelines/）                │
│  ConfigurablePipeline 读取 data/pipeline_configs.json         │
│    ├─ FactorRegistry  （src/factors/）计算特征                 │
│    ├─ TargetRegistry  （src/factors/）生成标签                 │
│    └─ ModelRegistry   （src/models/）训练并预测                │
│  输出 prediction {prob_bull, score, signal, ...}              │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                  Strategy 层（src/strategies/）               │
│  将 prediction 映射为 target_position（0.0 ~ 1.0）            │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                  Trade 层（src/strategy.py）                  │
│  calculate_trade_shares() 把目标仓位 → 按手对齐的买卖股数     │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                  Backtest 层（src/backtest/）                 │
│  ComboBacktester 滚动执行 Pipeline + Strategy                 │
│  ComboScanner    扫描所有 (Pipeline, Strategy) 组合           │
│  输出收益率、最大回撤、夏普、胜率、交易次数等指标             │
└─────────────────────────────────────────────────────────────┘
```

更细化的数据流：

```
CSV / Yahoo Finance
       │
       ▼
load_stock(symbol) ──► Pipeline.fit(df) ──► Pipeline.predict(row)
                              │                    │
                              ▼                    ▼
                   FactorRegistry /         Strategy.decide()
                   TargetRegistry /                │
                   ModelRegistry                 ▼
                                       target_position
                                               │
                                               ▼
                                    calculate_trade_shares()
                                               │
                                               ▼
                                       回测 / 实盘决策
```

## 3. 组件清单

### 3.1 因子（FactorRegistry）

| 名称 | 说明 |
|------|------|
| `sma` | 简单移动平均 |
| `std` | 滚动标准差 |
| `zscore` | 收盘价偏离均线的标准差倍数 |
| `slope` | 均线/价格变化率 |
| `returns` | 区间收益率 |
| `golden_cross` | 快线是否大于慢线 |
| `ma_gap` | 两条均线差距百分比 |
| `volatility_ratio` | 变异系数 |
| `macd` | MACD 指标（dif/dea/hist） |
| `rsi` | 相对强弱指数 |
| `bollinger` | 布林带（upper/lower/mid） |
| `rolling_tree_prob` | 滚动决策树上涨概率特征 |

### 3.2 监督目标（TargetRegistry）

| 名称 | 说明 |
|------|------|
| `ma_direction` | 未来 N 期均线方向 |
| `price_direction` | 未来 N 期收盘价方向 |
| `return_bucket` | 未来 N 期收益率是否超过阈值 |
| `volatility_breakout` | 未来波动率是否突破 |

### 3.3 模型（ModelRegistry）

| 名称 | 说明 |
|------|------|
| `dual_logistic` | 两颗 Logistic 分别预测 MA5/MA20 方向，加权综合 |
| `logistic` | 单颗 Logistic 预测未来方向 |
| `dual_random_forest` | 两颗随机森林分别预测 MA5/MA20 方向，正则化 |
| `random_forest` | 单颗随机森林 |
| `macd_rule` | MACD 金叉死叉规则模型 |
| `rsi_rule` | RSI 超买超卖规则模型 |
| `boll_rule` | 布林带上下轨规则模型 |

### 3.4 Pipeline（PipelineRegistry）

| 名称 | 内部模型 | 说明 |
|------|----------|------|
| `ma_dual` | `dual_logistic` | MA5/MA20 方向双模型 |
| `dt_logistic` | `logistic` | 决策树提取概率特征 + Logistic |
| `rf_ma_dual` | `dual_random_forest` | 正则化双随机森林预测 MA 方向 |
| `macd` | `macd_rule` | MACD 动量信号 |
| `rsi` | `rsi_rule` | RSI 阈值信号 |
| `boll` | `boll_rule` | 布林带反转信号 |

### 3.5 策略（StrategyRegistry）

| 名称 | 说明 |
|------|------|
| `prob_position` | 根据 `prob_bull` 线性映射仓位（0.3→0，1.0→满仓） |
| `low_risk` | 在 prob_position 基础上限制最大仓位（默认 10%） |
| `macd_cross` | MACD 金叉满仓、死叉空仓 |
| `rsi_threshold` | RSI < 30 买入，> 70 卖出 |
| `boll_reversion` | 触及下轨买入，触及上轨卖出 |

## 4. 决策流程

1. 读取 `data/best_combos.json` 获取该股票当前最佳 (Pipeline, Strategy)。
2. 加载历史日线，调用 `pipeline.fit(df)` 训练。
3. 对最新一天调用 `pipeline.predict()`，得到 `prob_bull`、`score`、`signal` 等。
4. Strategy 根据预测与上下文（开盘价、前收盘价、资金、手续费）生成 `target_position`。
5. `calculate_trade_shares` 把目标仓位转换为实际买卖股数（按手数对齐）。

---

## 5. 回测机制

`ComboBacktester` 对任意 (symbol, pipeline, strategy) 进行逐日滚动回测：

- 每天使用 `[0, i)` 的数据训练 Pipeline。
- 用 T 日开盘价成交。
- 支持回撤止损（组合层面回撤达到阈值时清仓）。
- 输出指标：总收益率、最大回撤、夏普、胜率、交易次数、最终资产、手续费。

## 6. 回测结果示例

### 6.1 通富微电 002156.SZ（近半年：2025-12-18 ~ 2026-06-17）

| Pipeline | Strategy | 收益率 | 最大回撤 | 夏普 | 交易次数 |
|----------|----------|--------|----------|------|----------|
| `ma_dual` | `prob_position` | **+40.83%** | -3.58% | 3.233 | 48 |
| `rf_ma_dual` | `prob_position` | +20.51% | -1.49% | 3.967 | - |
| `dt_logistic` | `prob_position` | +9.46% | - | - | - |

> 当前最佳组合：`ma_dual + prob_position`。Tree/Rolling 类模型在该区间未超越 Logistic 双模型。

### 6.2 组合扫描

`scan 002156.SZ --start 2025-12-18 --end 2026-06-17 --max-drawdown 5`

系统在满足最大回撤约束下选出收益最高的组合，并保存到 `data/best_combos.json`。

## 7. 配置管理

### 7.1 Pipeline 配置

`data/pipeline_configs.json` 保存每个 Pipeline 的完整配置：

- `factors`：使用的因子与参数。
- `model.type`：引用的模型名称。
- `targets`：训练目标。
- `feature_cols`：送入模型的列。
- `predictions`：模型输出到统一 prediction 键的映射。

### 7.2 最佳组合注册表

`data/best_combos.json` 保存每只股票的最佳组合及测试区间、指标：

```json
{
  "002156.SZ": {
    "pipeline_name": "ma_dual",
    "strategy_id": "prob_position",
    "pipeline_config_id": "ma_dual_v1",

    "metrics": {"total_return_pct": 40.83, "max_drawdown_pct": -3.58, "sharpe": 3.233, "trade_count": 48},
    "last_evaluated": "2026-06-19T..."
  }
}
```

## 8. 扩展方式

新增组件只需三步：

1. 实现类或函数（模型 / 因子 / Pipeline / Strategy）。
2. 在对应注册表中注册。
3. 在 `pipeline_configs.json` 或 CLI/Web 中引用。

具体代码模板见 `AGENTS.md`。

---

## 9. CLI 与 Web UI

- CLI 入口：根目录 `QuantCLI.bat <子命令>`，等价于 `scripts\strategy.bat <子命令>`。
  - 支持交互式候选提示：省略股票代码/Pipeline/Strategy 时自动列出可选项目。
  - 支持 `help` 子命令：`QuantCLI.bat help [COMMAND]`。
  - 支持 `--no-color` 禁用颜色输出。
- Web UI：根目录 `QuantWebUI.bat` 启动 Flask（`src\api_server.py`），调用与 CLI 相同的底层 API。
- 基本面分析：通过 CLI `strategy fundamental` 触发，Web 端只读展示。

## 10. 后续研究方向

1. 引入更多模型：XGBoost、LightGBM、神经网络等。
2. 增加目标类型：未来收益率分桶、波动率预测等。
3. 组合层面的仓位分配：多股票之间按夏普/回撤分配资金。
4. 盘中数据接入：5/15/30 分钟 K 线，优化 T 日执行价格。
5. 自动化调度：定时更新数据、定时运行分析并保存最佳组合。
