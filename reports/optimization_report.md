# T日决策说明

> 说明当前平台如何根据 T-1 收盘数据与 T 日开盘价生成交易建议。

---

## 1. 决策时机

平台默认在 **T 日开盘后** 进行决策，原因：

- A 股开盘价（09:30）已反映隔夜消息面、外围市场（如 AMD）走势与集合竞价结果。
- 实际交易中，T 日开盘价比 T-1 收盘价更适合作为当日成交参考价。
- 建议在北京时间 **09:35 ~ 09:45** 观察首根 5/15 分钟 K 线后确认方向。

## 2. 决策流程

1. 读取该股票已保存的最佳 `(Pipeline, Strategy)` 组合。
2. 加载全部历史日线数据，滚动训练 Pipeline。
3. 对最新一天调用 `pipeline.predict()`，得到上涨概率、信号、指标值等。
4. Strategy 根据预测结果与当日开盘价生成 `target_position`（目标仓位）。
5. `calculate_trade_shares` 按手数对齐，计算实际买卖股数。

## 3. 开盘价来源

优先级如下：

1. 用户手动输入（CLI `--open` / Web UI 输入框）。
2. 日线数据已更新到今日，则使用今日 `open`。
3. 否则使用 T-1 收盘价作为保守参考。

## 4. 策略对开盘价的处理

不同 Strategy 对开盘价的处理方式不同：

- `prob_position` / `low_risk`：开盘价仅用于成交价格计算，目标仓位由 `prob_bull` 决定。
- `macd_cross`、`rsi_threshold`、`boll_reversion`：信号由 Pipeline 规则模型产生，开盘价仅用于成交。

未来可在 Strategy 中扩展“跳空修正”逻辑，例如看多但高开过多时降低仓位。

## 5. 交易成本

回测与决策默认使用：

- 单笔手续费：5 元
- 每手股数：100 股（A 股）
- 滑点未单独建模，以开盘价近似实际成交价

## 6. 建议

- 强趋势行情：尽早按目标仓位执行，避免踏空。
- 震荡/高波动行情：结合开盘跳空与首根 K 线方向确认后再执行。
- 若早盘无法操作，可在收盘前按目标仓位执行作为 fallback。
